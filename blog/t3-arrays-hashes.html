Arrays and Hashes

What's the difference?

7/24/15

The most simplified definition of an array: an ordered list of objects. These objects can be integers or words(strings). Arrays allow the user to store and retrieve objects from the array. In order to retreive an object within an array you must call it by using integers. Here is an example of an array in Ruby:

names = ['Matt', 'Joe', 'Steve', 'Ben']

The array is called 'names' and the data stored inside are the four names within the brackets. To retreive the name 'Matt' from this array you would use the following ruby code:

return names[0]

When calling an array, the first object takes position 0. This is a very simple example of an array but it demonstrates how the user can store and retreive data. Now that you have a basic understanding of arrays we can move on to hashes

A hash is very similar to an array. A hash allows the user to store and retreive data using a key and value structure. Here is an example of some Ruby code:

id_number = { tim: 1, john: 2, pat: 3}

This simple hash called is called id_number, its keys are the names and its values are the integers. This should give you a very basic understanding of why hashes are useful when programming with Ruby. To retreive Tim's id number you would use the following ruby code:

id_number[:tim]

This code will retreive the number 1. Hopefully this blog post gives you a basic understanding of arrays and hashes.

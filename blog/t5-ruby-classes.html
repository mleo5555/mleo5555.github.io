Ruby Classes

8/9/15

Creating a class in Ruby allows programmers to group methods together into a single bundle. This is convenient because it allows us to quickly create objects that behave the exact same way. Every individual object created by a class is referred to as an "instance" of a class. Here is an example of a simple class in Ruby:

class Car

   def initialize(brand, name)

      @brand = brand

      @name = name

   end

   

   def vroom

      puts 'Vroooom!'

   end

   

   def details

      puts "This fine car is a #{@brand} #{@name}."

   end

end

Let's start from the top. A class is defined like this: class ClassName. In my example Car is the class which must be written in CamelCase. Within the class Car are 3 methods: initialize, vroom, and details. All three methods create the behaviors of the class Car. initialize is a class method while vroom and details are instance methods. Let's go over the differences between class methods and instance methods.

Class Methods vs. Instance Methods

Class methods are only called on the class name while instance methods are only called on instances of a class. This may sound confusing but I promise it is simple. Let's use some more code as an example:

my_car = Car.new("toyota", "camry")    The class method initialize is being ran on the class Car and accepts two arguments. Since my_car is a new instance of the Car class, instance methods may be called upon it.

my_car.vroom

will puts Vroooom

my_car.details

will puts This fine car is a toyota camry.

This brief explanation should give you a basic understanding of Ruby classes, class methods, and instance methods.
